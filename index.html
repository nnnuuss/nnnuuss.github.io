比较简单的实验帮助理解命令行是如何读取输入的,

argc是读取的操作符个数,比如输入sleep操作数就为1, sleep 10操作数就为2, argv数组是一个字符串数组,里面存的是输入内容,argv[0]等于执行进程的名字,例如这个例子,argv[0] = “sleep”, argv[1] = “10”;

Pingpong:
这个lab稍微难一点,起初我还没弄清楚,但查阅别的代码终于理解了这个lab的意义,首先在很多情况下,父进程进行读取之后,是需要将执行交给子进程的,而子进程与父进程使用的代码内存是相同的,但是又需要父子进程之间进行信息交流,那么这个地方就引入了管道(pipe)的概念,这个就是父子进程进行信息交流的关键,一个pipe有两端,读取端和写入端,管道一直是连通的,所以当你像一个两边都是可通行的管道中写入数据的时候,数据会外泄(也许?我猜的), 所以在进行写入的时候,要将读取的一端关闭,读取的时候,要将写入的一段关闭,后者是为了在以后从管道中读取时可以访问EOF,以便终止read函数.

题单链接:Dashboard - Codeforces Round #691 (Div. 2) - Codeforces

A题链接Problem - A - Codeforces

题目大意:一张卡片上有红蓝两个数字,对卡片进行全排列,如果全排列中红色数字大于蓝色数字的次数较大,则红色胜利,否则蓝色胜利,如果次数相同则平局.
题解:简单题,如果当前卡片红数字比蓝数字大,那么后续的排序中,一定存在这个位上红字大于蓝字,反之亦然,如果蓝字红字的贡献值相同的话,那么就是相等.

B题链接Problem - B - Codeforces

题目大意:给你一个点,给你任意步数,求在走完所有步数后,这个点可以到达的地方有多少个,重复位置不重复计算,初始时一个点可以四个方向都选择,但之后的每一步,如果上一步是上下走的,这一步只能左右走.
题解:先写了个dfs跑了一下,发现数据量太大,跑不完,然后打表,发现规律,直接用规律做了.

C题链接Problem - C - Codeforces

题目大意,给你两个序列a和b,对于b中的每一个元素,输出a序列中每个元素与b中元素相加得到序列的gcd.
直接查的blog,得到一个这样的规律,gcd(a, b) = gcd(a, b-a),向外推广则有gcd(a, gcd(b, c) = gcd(a, gcd(b, c-b)),所以对于序列a和b,gcdrow = gcd(a1+bj, a2+bj, ….. an + bj) = gcd(a1 + bj, a2 - a1, a3 - a2, an - an-1),然后求出gcd(a2 - a1, a3 - a2, an - an-1) = g,再求gcd(g, a1 + bj)即可.

题单链接:Dashboard - Educational Codeforces Round 105 (Rated for Div. 2) - Codeforces

A题链接Problem - A - Codeforces

大致题意:输入一串只有ABC组成的字符串,A,B,C各代表一个’(‘ 或 ‘)’, 问是否可以通过合适的转换,使得得到的括号序列合法.
比较麻烦的一道模拟题,我做了很久都没做出来,看了一下官方题解,主要思路是先判断头尾两个字符,头字符代表的一定是’(‘,尾字符代表的一定是 ‘)’,然后通过计算头字符和尾字符的个数判断最后一个字符代表那种括号,然后在对括号序列进行一次检验,当右括号大于当前左括号的时候就不合法.

B题链接Problem - B - Codeforces

大致题意:给你一个方阵,并告知你四个边上染色的方块数量,判断是否有这样的染色方式使得染色结果合法.
构造题,我开始想的思路是把四个点进行一一枚举,但是太麻烦,而且很难枚举到最优解上,看了别人的博客,学会了这样的思路,只有当边上染色个数等于边长度时,才有确定的染色块,如果等于边长度-1则有一不确定染色快,然后判断当前边上的染色快是否小于确定数量染色快,和对应两边染色块数量是否小于不确定与确定之和.

D题链接https://codeforces.com/contest/1472/problem/D

这里需要先考虑一个问题,如何才能使对方的分数尽可能地小于自己,一种方式是自己拿可以得分的,另一种方式是拿最大的数据,显然是第二种,因为拿了最大的数据,别人就无法得到这个分数,相当于自己获得了相应的分数,用map存储数据然后遍历一下.

map学到的一些使用规则,当想要指向最后一个元素的时候先让指针指向end,然后–,因为end是指向nullptr的,但又与最后一个元素相邻,所以需要先–,还有关于begin,begin是无法支持自减的,所以上面的判断还多重复了一次,虽然我也不想这么写,但确实没啥好办法.

收获:

一定要考虑basecase的情况,因为有些题目案例不提供单个输入的情况,提交前先考虑单个的特殊情况是否程序可以处理.

695div2B. Hills And Valleys

发表于 2021-04-09 分类于 每日刷题

B. Hills And Valleys

time limit per test

1 second

memory limit per test

256 megabytes

input

standard input

output

standard output

You are given a sequence of n integers a1,a2, …, an. Let us call an index j (2≤j≤n−1) a hill if aj>aj+1 and aj>aj−1; and let us call it a valley if aj<aj+1 and aj<aj−1.

Let us define the intimidation value of a sequence as the sum of the number of hills and the number of valleys in the sequence. You can change exactly one integer in the sequence to any number that you want, or let the sequence remain unchanged. What is the minimum intimidation value that you can achieve?

Input

The first line of the input contains a single integer t (1≤t≤10000) — the number of test cases. The description of the test cases follows.

The first line of each test case contains a single integer n (1≤n≤3⋅1e5).

The second line of each test case contains n space-separated integers a1, a2, …, an (1≤ai≤1e9).

It is guaranteed that the sum of n over all test cases does not exceed 3⋅1e5.

Output

For each test case, print a single integer — the minimum intimidation value that you can achieve.

因为改变一个值只有两种改变方式,成为左边值,成为右边值,每次改动对山峰和山谷造成的影响可能不同,所以需要从第二个一直到倒数第二个不断地进行左变换右变换这样去枚举,才能得到正确答案
