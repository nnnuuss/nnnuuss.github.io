Problem - C - Codeforces

Problem - A - Codeforces

题目大意:给定一个n*n的方格,对于每一行中最小的元素称为ai,集合S = {a1, a2, …an} ∩ {1, 2, …, n},求出所有排列方式的S大小的和.(S的大小指的是S中的元素个数).

思路:分别考虑1到n的每个数字对于答案的贡献,如果第一个方格选取1,那么该行剩余的部分就是从n^2-1个中选取n-1个元素,然后对于当前行可以进行一个全排列,对于剩余的元素进行一个全排列,然后对当前行再排列一下,乘n.对于求解的方法,需要进行化简,将分母化为只有一个乘积,然后直接求逆元来降低复杂度.

Problem - C - Codeforces

题目大意:现在给定一棵树,我们称优美的删除方式为删除若干个节点后,不存在孤立的节点,请你求出有多少种删除方法,如果删除的顺序不同而元素相同,视为一种方法.

思路:树形DP,给一个节点定义以下三种状态:1.该节点被删除;2.该节点保留且不存在节点与它相连接;3.该节点保留且存在至少一个节点与它相连接.对于状态的转移,第一种状态合法情况有两种:该节点的子树被删除,该节点的子树保留且至少存在一个节点与其连接.第二种状态就是由其子节点全删除转移过来,第三种状态就是由它所有子树状态转移过来然后减去第二种状态,顺着这个思路dp就可以了.

补一点自己的问题,因为第一次建树采用的是单向边,然后做了点的交换处理,后来发现不能这么做,因为一棵树你无法确定那个点是超级源点,所以建树一定要建双向边,然后遍历的时候做一个特判.

Problem - K - Codeforces

题目大意:现在给定n个点,m条边,以及各边的边权,然后有q此询问,每次询问需要计算原图在删除所有小于当前询问值qi的边之后,有多少对点是连通的.

思路:比较好的一道思维题,首先我们直接建图肯定会超时,那么我们思考一下能不能改变处理顺序,让建图变成一个一次性的过程,那我们把边按照权值大小排个序,询问值的大小也进行一个排序,然后一个指针遍历询问值,一个指针遍历边权,这样建图只用遍历一次就可以得到答案.图形的维护我们采用并查集,因为只需要维护连通性,那么并查集就是最好的选择,每次合并两棵子树的时候,先减去之前两个子树的贡献,然后再加上它两合并之后的贡献,用一个数组来维护以某个节点为根的子树节点个数.

题目大意:给定数p和q,请你找出最大的数x,使得x是p的因数,且q不是x的因数.

思路:数论题.如果p不能被q整除,那么q就不是p的因数,那么最大的数x=p.如果可以被整除,那我们只需要求出一个最小值,当p除以这个最小值时,q不再是p的因数即可.例如4是p的因数,也是q的因数,那么如果将p除以2以后,q就比p多了一个因数2,q就不再是p的因数了,所以我们需要做的就是找出p和q的所有因数,然后对每个因数进行枚举就可以了.
