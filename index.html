1.枚举

Flip Game - POJ 1753 - Virtual Judge (vjudge.net)

题目大意:给定一个黑白棋盘,每次可以选择翻动一颗棋子,当一颗棋子翻转后,其四周的棋子都会进行翻转,求如何最少次数的翻转可以使得棋盘上的棋子都为黑或者白.
思路:因为之前的枚举第一行的情况不知道为啥不过,所以重新写了一个二进制枚举所有棋子状态的,第一行就表示0000,这样16个数字由一个16位数字表示,1表示进行翻转.

2.贪心

Radar Installation - POJ 1328 - Virtual Judge (vjudge.net)

题目大意:x轴下部为陆地,上部为小岛,可以随意在陆地上布置炮台,问最少使用多少炮台可以使得所有的小岛都在炮台的射程内.

思路分析:首先为了保证炮台的射程尽可能地大,我们的炮台一定要放在x轴上,这样才能确保y轴的距离尽可能地小.
现在分析y轴该如何确定,我首先想的是把所有的小岛以x轴的位置进行一个升序排序,然后每个炮台都与最左边对其,但这样的思路显然是错,例如如果这样选择先跟A对其,那么B就不在范围内了.
所以正确的做法是,逆向思维,确定每个小岛的炮台区间,然后求最小无交集区间个数.

二,最短路径算法

Currency Exchange - POJ 1860 - Virtual Judge (vjudge.net)

题目大意:给定你若干种货币,给定你一个交换规则,你可以通过手上的货币与其他货币进行交换,每次交换得到的货币为(已有货币-手续费)*汇率,求解是否可以通过交换后,使得手上的货币数量和原来的相同.
思路:是一个图的问题,问题经过分解之后,就是在一个起点开始,求整个图是否有负圈,如果有负圈的话,那么就可以使得货币不断地增加,最总抵消手续的费用,所以把问题转化一下,就是在有限次数内求最短路径,如果所求的次数超过了有限次,那么一定存在负圈,也就是答案成立.因为是单源图,所以可以直接采用bellman,直接更新到达每个点的最短路径,然后不断更新.

昂贵的聘礼 - POJ 1062 - Virtual Judge (vjudge.net)

题目大意:每样物品有它的价值,和优惠价值,优惠价值就是当你使用这件物品和别人交换物品时需要付出的价值,每件物品的所有者还有等级制度,每次进行交易的等级差值不能高于题目所给条件.求交换到第一件物品所需要的最小价值.
思路:dijk求最短路,不过因为是每次连续的交易不可以超过m,所以要对每个节点进行一次dijk,然后得出答案,建图比较辛苦.

Problem - B - Codeforces

Problem - C - Codeforces

题目大意:给定一个n*n的方格,向里面填n^2个数,要保证每个数的上下左右差值都不为一,如果可以构造出这样的写法,那么就输出合理的符合的答案,如果不符合,就输出-1;
思路:构造题,直接斜着双向构造,斜着构造可以避免四个方向上差值为1,双向构造可以避免最后两列出现不符合的情况.当n=2的时候特判是无解的.

Problem - D - Codeforces

题目大意:在一个数组中,当两个数ai, aj满足i < j, ai-aj = i-j时称这样的一对ai,和aj为一组特殊数,求一个数组中有多少个特殊数对.
思路:ai - aj = i - j 等价于 ai -i = aj-j,只需要记录每次元素与下标的差值,然后存入最后读取差值相同的个数,就可以得到答案.

Problem - E - Codeforces

题目大意:给定几只羊和一定的空间,你需要通过一定的移动,使所有的羊都聚在一起,排成一列,求最小需要的操作次数.
思路:因为输出量1e6,所以必须线性解决,通过线性扫描中间点的方法来做,每次从1开始扫描直到n,让所有的羊以1为分界点,所有1左边的羊都聚集在这里,1右边的羊都聚集在右边,然后扫描一次得到答案,这里可以用前缀和,来保存某个分界点左边的羊的数量和羊所在的距离,右边同理,然后通过计算羊从起点开始分别移动到分界线的距离,减去羊本身已经移动的距离,就可以得到答案,右边则是同样的原理,不过要交换顺序.

Problem - B - Codeforces

题目大意：给定一个直方图，你可以通过操作来改变直方图的形状，每次可以选择一个直方减少一格，一个直方图的丑陋值为直方图的垂直边长加上操作数，求每个直方图的最小丑陋值。
思路：模拟加思维，如果一个直方同时高于左右两个直方，那么它每一格所带来的丑陋贡献值为2，而进行一次操作后，总丑陋值就减少了1，所以每次只需要把同时高于两边直方的直方减少到两边直方中较高直方的同一高度就可以了。

Problem - C - Codeforces

思路：并查集，因为如果要上下进行一次调整，那么跟它相关的元素也要进行调整，这样构成的一个集和都需要进行调整，每个集合都可以进行两次排列，只需要先用并查集进行构造就好。

Problem - D - Codeforces

题目大意:给定a, b, k三个数,你可以选择任意数n,保证a或者b可以被n整除,然后使a/=n,或b/=n,求能否使用恰好k次操作,使得a=b.

这里提一下,想出来了做法还不一定有用,这题时间卡的很紧,因为测试数据量不小,比较好的方式是写一个素数筛,但是如果只筛到1e5, 后面的质数有晒不掉的可能,所以需要特判当前数是否为1,如果不是那么就是质数.反正这道题比较怪,用素数筛需要加特判,如果直接硬枚举得话,时间卡的紧,但是优化优化也能过.

Problem - F - Codeforces

Problem - G - Codeforces

题目大意:给你四个数a和b,还有c和d, 求a和b最多能分成多少组c和d,每次选择数字组成c和d时只能选择a或者选择b,不能混合选.
思路:二分答案+贪心,二分答案还比较好想,因为主要是个解方程的问题,但是具体怎么判断是向左还是向右取边界,就是贪心的思想了,首先,如果cd中较小的那个数满足当前二分出的答案的话,那么就先将a和b都替换成每组c个元素,然后再将若干个c组合并成d组,此时替换过程中求出的就是已满足c的情况下d尽可能大的个数,如果d的个数满足当前二分答案的话直接右移即可.

给定两个数A, B.确定能否找出三个不同的数x, y, z满足x + y = z, 且三个数一个要能被A*B整除,另外两个要只能被A整除, 如果找不出来则输出NO.

Problem - B - Codeforces

题目大意:给定一个长度为n的数组,你可以对它进行至多n次变化,在变换后,需要保证数组中任意一对相邻元素互质.变化遵循这样的规律,低于数组中的元素ai, aj, 和你选择的数x, y,当满足min(ai, aj) = min(x, y)时, ai = x, aj = y.

Problem - C - Codeforces

题目大意:如果一个字符串仅由01组成,且相邻字符都不相同,那么就称这样一个字符串为不稳定串,如果一个字符串由01?组成,且通过将?替换成0或1的话,那么这样的一个串就称为beautiful串,对于给定的字符串,求出最大数量的beautiful子串.

A题Problem - A - Codeforces

思路:是一个构造题,主要考查一个思维,因为给的序列的元素个数一定是偶数,那我们这样考虑,当整个序列满足较小较大较小这样的顺序来进行排列的话,那么一定可以满足题意.

B题Problem - B - Codeforces

题目大意,给你一个数,判断这个数能否被11, 111, 1111等等组合而成.
思路:思维题,因为1111 = 11 * 100 + 11, 11111 = 111 * 100 + 11.所以后续的数字都不用考虑,只需要考虑11和111,是否满足11x+111y = n,且x,y都为整数,这里可以直接的做法是枚举xy的值,但是n很大必然会超时.
仔细观察一下我们可以发现111 = 11*10 + 1,所以只需要n%11的余数小于等于11的个数,那么就是符合的.即n/11 >= (n % 11) / 10

C题Problem - C1 - Codeforces

题目大意:给定一个序列,序列中的值有正有负,初始值为0,到达某个位置时当前值会与位置上的值相加,求在保证任何时刻都不为负数时的最大通过位置数.
解题思路:优先队列,如果是正数,那么一定加入状态值当中,如果是负数,且相加之后大于等于0,那么先用优先队列存储这个选择,然后在看后面的决定,如果当前相加大于等于0,且队列中是有选择存在的话,就需要进行一次抉择,如果当前元素大于队列中的最小元素,那么最小元素出列,当前元素进列,然后继续维护队列中的最小值.

A题Problem - A - Codeforces

题目大意:给定一个数x,求出一个数y使得x&(x-1)&(x-2)&…&y = 0

做题思路:首先这是一道位运算的题目,为了使x与其他&运算为0,则x上每一位为1的数都应该存在一个在该位上为0的数来进行运算,x的最高位一定为1,那么y最高位必须低于x一位,在当前情况下y的最优解就是其他位上均为1,这样无论如何都能得到一个除最高位为1其他位都为0的数.例如10111和01111,01111 + 1 = 10000,这样经过运算之后就可以得出0.

B题Problem - B1 - Codeforces

题目大意:给定一个只由0和1组成的回文串,让ALICE和BOB对这个串进行操作,有两种合法操作,一种是让其中的一个0变成1,会花费一元钱,还有一种是操作是将串反转,不花费任何费用,且如果该串此时为回文串时不可以进行反转,或者上一轮刚反转过这轮不允许反转,当字串全部变为1时游戏结束,花费少的胜利.
做题思路:是一个分析情况的题目,需要从0的个数开始分析,如果是偶数个0,那么ALICE先手,只能先进行一次转换,然后BOB就可以将串进行反转,如果该串只有2个0,那么BOB就可以获得2分的领先,如果0的个数大于2,BOB至少能获得1的领先,所以可以得出结论,偶数0谁后手谁赢.
如果是奇数个0,那么必定有一个0在中间,ALICE就可以先手把这个0转换掉,然后自己变成后手,0的个数又为偶数,那么ALICE就必定获胜.
补充一下为什么存在偶数0后手一定剩的原因,当只剩下2个0的时候,后手只需要进行一次反转,就可让先手必须进行两次转换,而当0的个数大于2的时候,先手如果将串变成分回文串,后手只需要将串再变为回文串就可以保持原状态继续进行下去.所以偶数0后手必胜.
还有特殊情况,就是只存在一个0的时候,这时候ALICE必须进行操作,所以BOB获胜.

B2题Problem - B2 - Codeforces

题目大意:B题的升级版,提供的串初始状态不一定是回文串.
一直没想出来该怎么算,看了别人的视频才知道的,不得不说别人真的牛.
这里给一个上一题没讲的胜利关键条件,两人都需要尽可能的把串还原成回文串,然后再通过调整自己的先后手,来达成胜利.
解题思路:这道题需要一定构造案例的能力,我们还是分类讨论,当给的串为回文串的时候,那么同easy版一样,当给定的串为非回文串的时候,那么差异的元素一定是0,如果差异0的个数大于等于3的话,ALICE前三次只需要进行反转就可以取得3分的领先,这样的话即使让出后手也可以取胜,如果差异0的个数等于2的话,ALICE先获得2分的领先,然后先手进行,如果当前0的个数为奇数,根据前面的思路ALICE必胜,如果0的个数为偶数的话,ALICE只需先反转然后再将最后一个差异0变成1,就得到了偶数个0的后手情况,ALICE必胜.
当差异0的个数为1的时候,如果0的个数为2,那么就是平局,因为此时串一定是奇数,且中间位置为0,ALICE直接把串变为回文,然后各得一分得到平局的结果,如果0个数大于2,那么如果0为偶数又可以得到奇数0先手,0为奇数则可以得到偶数0后手,ALICE必胜.

C题Problem - C - Codeforces

题目大意:给定你一个数字串,求这个串所能构成的所有子串的序列数.
序列:当两个元素值相同且顺序(下标)不相同时即构成一个序列.
子串:sequence连续的一段子序列(substring子序列可以是按顺序但不连续的)
因为当时根本就没读懂题,以为是子序列来写,瞬间就懵了,看了别人的题解理解了思路.核心思想还是一个前缀和.

画了个草图来解释一下,因为子序列是在一段序列的前后加减,那么当我们找到两个相等的元素的时候,从前开始进行有a1种选择方式,从后添加又有R-a2+1种方式,我们可以发现,方式的个数就是下标,而且当一个元素已经出现了之后,后面的元素是可以与它进行累加处理的,例如现在有一个a3和a1,a2的值都相等,那么它们都要和a1,a2分别组合一次,因为r-a3+1都是相等的所以结果就是(a1+a2)*(R-a3+1).于是就可以得出这样的代码.

专题一:简单搜索

发表于 2021-05-15 更新于 2021-05-20

1棋盘问题 - POJ 1321 - Virtual Judge (vjudge.net)

类似于八皇后问题,使用回溯算法,然后对当前状态进行搜索.因为每颗棋子行列都不能重合,所以我们直接一行一行的选取列,这样只用考虑列是否重合.

2Dungeon Master - POJ 2251 - Virtual Judge (vjudge.net)

是一个三维搜索的题目,没有多大的难度,主要是写的时候要注意细节,因为代码量很大,我之前就因为一些细小的bug卡了一天,真是可恶,ac前还因为一个小细节TLE了,这题目真的很搞人.

3Catch That Cow - POJ 3278 - Virtual Judge (vjudge.net)

给定一个起点和一个终点,需要从起点到达终点,一共有三种移动方式,左移一格,右移一格,或是跳转到当前位置两倍的地方.
刚开始读题的时候用dfs写,觉得很简单易懂,结果果然超时了,看了别人的思路说用bfs写,以后这种最小步数的一般都考虑bfs.

4Fliptile - POJ 3279 - Virtual Judge (vjudge.net)

题目大意给定一个0,1组成的棋盘,每翻转一次棋格,格子四周的格子都会跟着翻转,求是使所有格子都变成给0的最小反转次数.
不是很会,查了别人的blog,说是二进制枚举,枚举第一行的状态,然后向下依次来进行操作,因为上一行的1只能通过下一行来进行翻转,所以只需要枚举第一行的状态就可以得到该种方式的情况.

11迷宫问题 - POJ 3984 - Virtual Judge (vjudge.net)

也是简单的模板题,唯一需要注意的是每个节点应该保存当前的路经,我是采用vector加pair保存的,然后直接bfs就可以了,不得不说poj真的毒瘤,C++11的标准全不支持,我也是服了.

只留C++11标准下的代码

12Oil Deposits - HDU 1241 - Virtual Judge (vjudge.net)

白书上面比较经典的水坑题了,主要巧妙在通过修改地图来达到标记的效果,广搜也可以这样从而节省空间,很标准的dfs.

14Find a way - HDU 2612 - Virtual Judge (vjudge.net)

无语了,这题好多坑,思路是两次bfs,然后把每个目标的步数存起来,最后遍历一起求解,而且一个人还不能穿过另一个人,真的无语了.
