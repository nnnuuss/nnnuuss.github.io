C - Not Equal (atcoder.jp)

题目大意:给定长度为n的序列,对于序列上的每个数字,你可以选择小于等于当前数字的任意一个数字,从而来构成一个新的序列,新的序列要满足没有重复元素,求有多少种排列方式.

思路:首先这是一个排列的题目,对于原序列上的数字,数字代表几,就表示当前位上可以选择的元素个数,根据排列的计算公式,n(n-1)(n-2)(n-3)…*1, 这是对于每个位上都有n种可能地结果,对于这题,我们需要对原序列进行一个排序,因为较大的数字,其多出来的元素无法被较小的数字进行选取,而较小的数字是可以被较大数字选取,所以要先对较小的情况进行处理然后再处理将大的情况,再根据公式求解.

D - Collision (atcoder.jp)

题目大意:给定N个城镇,各城镇之间总共有N-1条路,每条路的长度都相同.现在给出甲乙两人所在的城镇,甲和乙分别从自己的城镇出发前往对方的城镇,两人都按照最短距离来行进,求两人最终是在城镇上相遇还是路上相遇.

思路:首先这是一个树上问题,因为n个节点只有n-1条边,所以该图一定是一棵树,其次对于两者相遇在哪,实际上是求两城镇之间的最短距离是奇数还是偶数,因为树上的任意两点路径是唯一的所以也就是求该路径的长度.因为输入规模较大,因此对每次输入都进行一次bfs肯定会超时.

所以我们换一种角度来考虑,因为每条路的权值都相等,且图中不存在环,所以可以直接选取一个节点对整个图进行染色,对于相同颜色的节点,那么它们之间的最短距离一定是偶数,所以在城镇相遇,颜色不同则是路上相遇.

Problem - A - Codeforces

题目大意:给定两个数,你可以对两个数进行两种操作,两个数同时加1或者两个数同时减1,求两个数经过若干次操作后的gcd,并求出最小的操作次数.

思路:无论进行多少次操作,两个数的差值都不会改变,那么这个差值就是两个数的gcd,然后只需要使得两个数进行操作后都是差值的倍数就可以了,至于进行多少次操作,只需要求出分别讨论加1和减1的情况,然后取最小值.

Problem - B - Codeforces

题目大意:给定若干个车道,对于车道上的车辆有一定的阻塞值,阻塞值的算法为当前车道上车数-该车道以后每个车道上的车数的绝对值之和,你被允许进行若干次的车辆交换,使得车辆到达不同的车道上,求解最小阻塞值.

思路:为了使阻塞值尽可能地小,那么就应该使得每个车道上的车辆尽可能地数目接近,那么其实问题直接分解成了一种情况,任意两个车道上的车辆数目差值不是1就是0,那么最小阻塞值就是当前为1的数目乘上当前为0的数目.

Radar Installation - POJ 1328 - Virtual Judge (vjudge.net)

思路分析:首先为了保证炮台的射程尽可能地大,我们的炮台一定要放在x轴上,这样才能确保y轴的距离尽可能地小.
现在分析y轴该如何确定,我首先想的是把所有的小岛以x轴的位置进行一个升序排序,然后每个炮台都与最左边对其,但这样的思路显然是错,例如如果这样选择先跟A对其,那么B就不在范围内了.
所以正确的做法是,逆向思维,确定每个小岛的炮台区间,然后求最小无交集区间个数.

Currency Exchange - POJ 1860 - Virtual Judge (vjudge.net)

题目大意:给定你若干种货币,给定你一个交换规则,你可以通过手上的货币与其他货币进行交换,每次交换得到的货币为(已有货币-手续费)*汇率,求解是否可以通过交换后,使得手上的货币数量和原来的相同.
思路:是一个图的问题,问题经过分解之后,就是在一个起点开始,求整个图是否有负圈,如果有负圈的话,那么就可以使得货币不断地增加,最总抵消手续的费用,所以把问题转化一下,就是在有限次数内求最短路径,如果所求的次数超过了有限次,那么一定存在负圈,也就是答案成立.因为是单源图,所以可以直接采用bellman,直接更新到达每个点的最短路径,然后不断更新.

混合背包的模板题

混合背包类型问题,主要是对于每种物品,有数量限制,做法是直接把每种物品的若干数量展开,当作完全相同的不同物品来进行抉择,但是考虑到这样的时间复杂度直接从nw增加到nw∑k,时间复杂度过高,所以需要进行优化.因为nw的时间复杂度无法再继续优化,所以只能从每次抉择的物品上进行取舍,因为k1,k2和k2,k3效果是一样的,所以我们采取二进制分组的方式,使数量由2倍增,及1, 2, 4, 8…..然后最后剩下的一部分又作为一组进行抉择,这样就把k相关的复杂度降低到log了.

思路:ai - aj = i - j 等价于 ai -i = aj-j,只需要记录每次元素与下标的差值,然后存入最后读取差值相同的个数,就可以得到答案.

Problem - E - Codeforces

题目大意:给定几只羊和一定的空间,你需要通过一定的移动,使所有的羊都聚在一起,排成一列,求最小需要的操作次数.
思路:因为输出量1e6,所以必须线性解决,通过线性扫描中间点的方法来做,每次从1开始扫描直到n,让所有的羊以1为分界点,所有1左边的羊都聚集在这里,1右边的羊都聚集在右边,然后扫描一次得到答案,这里可以用前缀和,来保存某个分界点左边的羊的数量和羊所在的距离,右边同理,然后通过计算羊从起点开始分别移动到分界线的距离,减去羊本身已经移动的距离,就可以得到答案,右边则是同样的原理,不过要交换顺序.

Problem - A - Codeforces

题目大意:如果一个数组,对于数组中任意两个数,它们差的绝对值都在数组中出现,那么这个数组被称为nice,给定一个数组,你可以向其中添加元素,判断该数组通过添加后是否能够成为一个nice数组.
思路:首先我们分析不可能的情况,当出现负数的时候,就需要源源不断地向数组中添加数据,所以当数组中有负数的时候一定是不成立的;题目中又说最终数组中的元素不会超过300个,所以直接暴力模拟就可以了.

Problem - B - Codeforces

题目大意:给定一个字符串,求字符串中未出现的长度最短,字典序最小的字符串.
思路:直接暴力就行,直接构造子串然后在原序列中找就可以了,因为这题数据水了,所以三个字符就能过,但实际上可能存在的序列是有1000*1000个的,所以应该是需要26^5,即 5层循环来进行判断的.

Problem - C - Codeforces

题目大意:给定一个字符串,字符串由D和K组成,每个字符串的比值为该字符串D的数量比上K的数量,你可以对一个字符串进行分割,如果分割后得到的字符串比值相同,求对于一个字符串的每一位而言,最多能分成几份.
思路:是个思维题,如果你读取到一个点,这个点的比值与之前读取到的某个点的比值相同,那么该串一定是从那个点之后又形成了一个子串满足同样的比值,主要问题在于怎么模拟,细节在代码注释中.

Problem - A - Codeforces

题目大意：给定一个地图，地图上有白色，红色，和未涂色的区块，询问是否可以经过涂色之后，使得白色区域和红色区域互不相邻。

Problem - B - Codeforces

题目大意：给定一个直方图，你可以通过操作来改变直方图的形状，每次可以选择一个直方减少一格，一个直方图的丑陋值为直方图的垂直边长加上操作数，求每个直方图的最小丑陋值。
思路：模拟加思维，如果一个直方同时高于左右两个直方，那么它每一格所带来的丑陋贡献值为2，而进行一次操作后，总丑陋值就减少了1，所以每次只需要把同时高于两边直方的直方减少到两边直方中较高直方的同一高度就可以了。

Problem - C - Codeforces

题目大意：给定若干个数的两组数据，你可以对数据进行上下行交换，但是不能让两组数据在同一行中出现相同数字，求满足条件的总排列方式。
思路：并查集，因为如果要上下进行一次调整，那么跟它相关的元素也要进行调整，这样构成的一个集和都需要进行调整，每个集合都可以进行两次排列，只需要先用并查集进行构造就好。

发表于 2021-06-13 更新于 2021-06-14 分类于 每日刷题

Problem - A - Codeforces

题目大意:给定一个数组,你每次可以从左边取一个数字,或者从右边取一个数字,求取得序列中最大值和最小值所需的最小次数.
思路:简单模拟就是了,一共有三种取法,从左向右取完,从右向左取完,两边都取,主要用swap来简化过程.

Problem - B - Codeforces

题目大意:给定一个序列,你可以选取任意k个元素,将它们的数字个数,任意的分给所有的元素,求出最小的k使得序列的每个数都相等,如果不存在k输出-1.
思路:直接求平均值就可以,当无法被整除的时候,就是无解的情况.大于平均值的数字都会被进行分发.

Problem - C - Codeforces

题目大意:给定一个序列,求出有多少组数对ai, aj满足i < j 且 l <= ai + aj <= r.
思路:读题的时候没看懂,直接跳了,以为是要求满足顺序下的数对个数,直接卡住,后来发现i和j大小关系的限制,只是为了防止数对重复出现一次.比较简单的题目,二分加排序,先把数组排个序,对于序列中的每个数,二分查找l-cur和r-cur+1的边界,两者直接相减就可以得到对于某个数符合的个数.

Problem - D - Codeforces

题目大意:给定a, b, k三个数,你可以选择任意数n,保证a或者b可以被n整除,然后使a/=n,或b/=n,求能否使用恰好k次操作,使得a=b.
思路:直接求两个数的最大可能的操作数,就是求a,b的质因数个数,如果k大于这个质因数个数,那么无论如何也无法恰好取得k,但如果k<质因数个数,只需要将若干个质因数相乘放在一次操作里,就可以对操作次数进行压缩.题目还需要对k=1的情况进行特判,如果两数相等一定不符合,如果一个数可以被另一个数整除,那么是符合答案的.
这里提一下,想出来了做法还不一定有用,这题时间卡的很紧,因为测试数据量不小,比较好的方式是写一个素数筛,但是如果只筛到1e5, 后面的质数有晒不掉的可能,所以需要特判当前数是否为1,如果不是那么就是质数.反正这道题比较怪,用素数筛需要加特判,如果直接硬枚举得话,时间卡的紧,但是优化优化也能过.

Problem - E - Codeforces

题目大意:模拟题,有两种语句,一种是给定一个变量,对它进行赋值,另一种语句是将两个变量拼接得到第三个变量,求所有变量中包含子串”haha”的个数,每个串只由’h’和’a’组成.
思路:模拟,真的就是模拟,不过直接模拟存不下这么长的字符串,或者你用python写好了交也可以,所以这里需要进行状态压缩,每次拼接只需要保留上次串的前三个和后三个,因为只要拼上一个字符就有可能影响”haha”串的数量,中间的串就可以直接压缩掉,模拟的细节就是需要把几个功能分开写,不然容易出错,(反正我是抄的).

Problem - F - Codeforces

题目大意:给定a和b,求从a增加到b,有多少个字符发生了改变,例如19就改变了8次,910就改变了2次.
思路:其实有点像找规律,从9到10一定是有额外一次,9x到1xx也是有额外一次,也就是说每出现一个10就会额外产生一次,100也是一样,只需要算出a和b分别额外的次数,再一相减就可以得出答案.

Problem - G - Codeforces

题目大意:给你四个数a和b,还有c和d, 求a和b最多能分成多少组c和d,每次选择数字组成c和d时只能选择a或者选择b,不能混合选.
思路:二分答案+贪心,二分答案还比较好想,因为主要是个解方程的问题,但是具体怎么判断是向左还是向右取边界,就是贪心的思想了,首先,如果cd中较小的那个数满足当前二分出的答案的话,那么就先将a和b都替换成每组c个元素,然后再将若干个c组合并成d组,此时替换过程中求出的就是已满足c的情况下d尽可能大的个数,如果d的个数满足当前二分答案的话直接右移即可.
