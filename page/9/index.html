参数传递与返回值

发表于 2021-03-11 更新于 2021-03-13 分类于 C++笔记

函数被定义出来,有时候需要对其进行参数的传递,才能保证函数的正常作用,那么什么样的方式才是参数传递的最优解呢,有下面几种优先级的判断:

向函数中传参的时候优先考虑以引用的方式进行传递,引用相当于一个封装好的指针,无论传的数据类型占多少字节,引用都只会传四个字节进去,即记录数据存在的地址,所以在当数据类型非常大的时候,用引用的方式传递会有很好的效果.

对于不需要对数据本身就行修改的函数,应养成良好习惯,在函数的引用传递时用const进行传递.

从编译器的角度来解读这个问题,当一个函数不以const进行传递的时候,这个函数内部是可能存在对引用数据的修改的,即使你在函数体内部没有进行这样的修改,但编译器无法得知你的函数是否进行了相关操作,当你传递一个常量对象来调用这个函数的时候,编译器只能报错.

Tips:

引用传递的实质上传的是一个四个字节的指针,如果你的函数需要单独的传一个字符,或者大小小于一个的指针的数据类型,那么传值的效率可能会比传引用高,但为了减少这些思考,尽量优先考虑pass by conference(to const).

上面讨论了参数传递的几种方式,下面我们来思考一下返回值的问题,延续之前的思路,当数据类型较大的时候,以引用的方式返回结果,效率会很高,那么是不是返回值也都应该以引用的方式进行返回呢,这里需要分情况讨论,因为以引用返回在有些时候是不适用的.

C. Minimum Ties

time limit per test

1 second

memory limit per test

256 megabytes

input

standard input

output

standard output

A big football championship will occur soon! n teams will compete in it, and each pair of teams will play exactly one game against each other.

There are two possible outcomes of a game:

  • the game may result in a tie, then both teams get 1 point;
  • one team might win in a game, then the winning team gets 3 points and the losing team gets 0 points.

The score of a team is the number of points it gained during all games that it played.

You are interested in a hypothetical situation when all teams get the same score at the end of the championship. A simple example of that situation is when all games result in ties, but you want to minimize the number of ties as well.

Your task is to describe a situation (choose the result of each game) so that all teams get the same score, and the number of ties is the minimum possible.

Input

The first line contains one integer t (1≤t≤100) — the number of test cases.

Then the test cases follow. Each test case is described by one line containing one integer n (2≤n≤100) — the number of teams.

Output

For each test case, print n(n−1)/2 integers describing the results of the games in the following order: the first integer should correspond to the match between team 1 and team 2, the second — between team 1 and team 3, then 1 and 4, …, 1 and n, 2 and 3, 2 and 4, …, 2 and n, and so on, until the game between the team n−1 and the team n.

The integer corresponding to the game between the team x and the team y should be 11 if xx wins, −1−1 if y wins, or 0 if the game results in a tie.

All teams should get the same score, and the number of ties should be the minimum possible. If there are multiple optimal answers, print any of them. It can be shown that there always exists a way to make all teams have the same score.

题目大意:存在n个队伍,每两个队伍要进行一次比赛,比赛的结果有两种情况

1.双方平局,则各得一分.

2.一方获胜,获胜方获得3分,另一方不得分.

求如何能以最少的平局情况下使得所有队伍的分数都相同.

考点:思维,数学.

解题思路:首先我们看题目的给的数据规模可以得知,数据规模较小,可以直接模拟来做,我们用一个数组来存储比赛的结果,数组的下标表示两边的队伍,那么当一个队伍赢了之后,与它进行比赛的队伍就要输,即当i对j赢了,j对i就输了.同时为了保证平局的次数最小,我们分两类情况来考虑:

1.每个队伍的对手为偶数个.

这种情况只需要每个队伍输赢各占一半就可以完成目标,不需要平局的情况.

2.每个队伍的对手为奇数个.

这种情况只需要每个队伍平局一场便可以成为第1种情况.

C. Pekora and Trampoline

发表于 2021-03-10 更新于 2021-03-13 分类于 每日刷题

难度:div2 rating:1700

time limit per test2 seconds

memory limit per test256 megabytes

inputstandard input

outputstandard output

There is a trampoline park with n trampolines in a line. The i-th of which has strength Si.

Pekora can jump on trampolines in multiple passes. She starts the pass by jumping on any trampoline of her choice.

If at the moment Pekora jumps on trampoline i, the trampoline will launch her to position i+Si, and Si will become equal to max(Si−1,1). In other words, Si will decrease by 1, except of the case Si=1, when Si will remain equal to 1.

If there is no trampoline in position i+Si, then this pass is over. Otherwise, Pekora will continue the pass by jumping from the trampoline at position i+Si by the same rule as above.

Pekora can’t stop jumping during the pass until she lands at the position larger than n (in which there is no trampoline). Poor Pekora!

Pekora is a naughty rabbit and wants to ruin the trampoline park by reducing all Si to 1. What is the minimum number of passes she needs to reduce all Si to 1?

Input

The first line contains a single integer t (1≤t≤500) — the number of test cases.

The first line of each test case contains a single integer n (1≤n≤5000) — the number of trampolines.

The second line of each test case contains n integers S1,S2,…,Sn (1≤Si≤109), where Si is the strength of the i-th trampoline.

It’s guaranteed that the sum of n over all test cases doesn’t exceed 5000.

Output

For each test case, output a single integer — the minimum number of passes Pekora needs to do to reduce all Si to 1.

题目大意:有n个蹦床,每个蹦床有一个数据a[i],每次落在一个蹦床上之后,会被弹到a[i] + i的位置,同时a[i]的值会减小一,直到弹出的距离外没有蹦床,这一次弹跳结束,求最少需要多少次弹跳.

考点:贪心, dp;

解题思路:首先为了保证都能减少到1,则要从输入的数据中值不为1的开始进行弹跳,直到该值减为1为止,采用模拟的方式来模拟弹跳步骤,时间复杂度过高,超时.所以这里需要利用动态规划的方式,来进行状态的转移.

首先我们用数组a来存储弹跳的距离,再用数组c来存储该蹦床被踩过的次数,当一个蹦床的弹跳距离不为1的时候那么他会在区间[i+2,min(蹦床数量, i + a[i]]产生贡献值,即这些蹦床都会被踩到,然后再往后迭代进行遍历,当一个蹦床被踩得次数大于蹦床本身的弹跳距离之后,先进行之前的贡献值计算,然后将多余的次数转移到i+1处,一直迭代下去,直到结束.计算次数值时,只需要遍历两个数组,用蹦床本身的弹跳距离减去被踩过的次数就可以,当踩过次数过多时,不能取复数,要以0为准.max(a[i] - s[i] - 1, 0).
