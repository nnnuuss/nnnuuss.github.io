Problem - A - Codeforces

题目大意:有三种尺寸的披萨,6,8,10,每种披萨分别需要花15, 20, 25块钱,有n个小孩,确保每个小孩至少都能有一块披萨吃,请问至少需要花多少钱.

思路:思维题,分类讨论即可.我们可以发现对于6,8,10所有尺寸的披萨,他们的价格都是5/2每块.当n<=6的时候,都至少需要一大小为6的披萨,当n>6之后,如果n是偶数,那么一定可以由6,8,10组合得到,那么只需要将n*(5/2)就可以,如果是奇数,那么将其+1变成偶数,然后运算即可.

Problem - B - Codeforces

思路:思维题,如果第二块木板放得下,那么必须第一块木板的长加上第二块木板的长小于平面的长,或者是宽满足对应关系也可以.那么我们就可以知道,最小的平移距离就是左右移动和上下移动,然后只需要模拟一下即可.

Problem - C - Codeforces

题目大意:给定一个2*n的地图,Alice和Bob从(1,1)处出发,需要到达(2,n)的位置,总共可以有两种移动方式,向下移动或者向右移动,每个方格上有一定的数值,Alice走过这些方格之后,数值就会被清零.现在Alice先走一次,然后Bob再走一次,请问Alice如何走才能使得Bob的得分尽可能地小.(Bob也会采取最优策略)

思路:思维+暴力,因为每个人都只有一次向下走的机会,那我们枚举Alice向下走的位置i,对于Bob而言,最优的策略只由两种,一种是从一开始就走下面一条路,这样他的的得分就是下面一条路的前i个,或者一开始就走上面,这样的话他的得分就是(i+1,n)这些方格的数值,枚举取最优即可.

题目大意:给定一个数n,Alice和Bob每次可以对这个数进行两种操作

  1. 将当前数除以该数的一个奇因数.

  2. 将当前数减1(该数为1时不可再进行操作).

Alice先手,如果谁无法进行上述操作了,那么该玩家失败.

思路:找规律,如果当前数是奇数,那么可以直接将该数除以它本身,所以当前数是奇数,先手一定获胜.

如果当前数是偶数,非三类讨论,如果当前数是2,那么一定后手胜,属于一个特判.

如果当前数的因数全为偶数,那么先手只能进行操作2,后手状态转移到奇数状态,后首胜.

如果当前数的因数有奇数,那么先手的赢法就是将该数的变换成非2的偶数,这样后手只能进行操作2,回到奇数状态,先手获胜.

Problem - C - Codeforces

题目大意:两个人交替参加比赛,共10场,当每场比赛的结果是1的时候就代表参加这场比赛的人获得1分,如果是0则不得分,如果是?则表示待定,现在两人进行比试,请你站在裁判的角度考虑至少比到第几场时,不需要再比了.(至少的意思是:如果当前两人的分差无法通过后续的比赛来进行弥补,那么就不需要再比了,同时你也可以使得?变成0或1来尽快结束)

思路:首先是个对于比赛结果不确定的情况,我们只由两种取法

一种是让奇数场次的不确定场全部获胜,偶数场次的不确定场全部失败

一种是让奇数场次的不确定场全部失败,偶数场次的不确定场全部获胜

只有这样分差才尽可能地大,然后对于结果的判断,只需要统计两个人的分数,然后判断当前的分差是否大于剩余场次能能拉平的分差,以此来判定是否提前结束比赛.

Problem - D - Codeforces

题目大意:你可以操控键盘打出一串预定字符,但在你准备输入一个字符时,你可以选择输入该字符,或者按下删除键(这样你不仅不会输入该字符,还会删掉之前存在的一个字符),现在给你一串预定字符,和一串目标字符,请问你是否可以打出目标字符串.

思路:思维题,首先如果目标串长度大于初始串那么一定不可能.

然后需要讨论两种情况

  1. 目标字符串和源字符串的差值为奇数.
    因为每次删除只由在开头的时候删除,才能删除一个字符,所以如果差值为奇数,那么一定是在开头就进行了一次删除.然后情况就变成了偶数情况

  2. 差值为偶数
    在后续的删除中每次删除就相当于把字符串同时删去了两位,所以对于这种情况只需要线性匹配一次就可以了.

得知这两种情况之后,其实只需要对两个串进行一次匹配,如果失配的话,直接将源字符串左移两位即可,相当于在某个字符失配之后直接进行删除.

Problem - A - Codeforces

题目大意:给定长度为n的排列,每组排列有山峰数k,请你进行构造使得n的排列恰好有k个山峰.

模拟题:首先山峰的最大数量为(n-1)/2,如果超过这个数那必然无法构造出来,然后再根据k的数量,从大到小放置n, n-1, n-2,…就可以了.

Problem - B - Codeforces

题目大意:对于一个序列a1,a2,….an,其中对任意一个j而言,满足a1&a2&…&aj = aj+1&aj+2&…..&an,我们称这样的序列是一个good序列,下面给你一个序列,请问该序列有多少种排列方式,使得整个序列是一个good序列.

思路:首先要明确的一点就是,为了使对于任意j,左半边和右半边的&运算结果都相等,那么一定有两个相同且所有位上匹配值最低的数放在两边,(这里的匹配值指的是所有位上,1尽可能地少,与其他数尽可能不重叠,这样的话才能保证无论和谁进行&运算,运算结果一定是它本身),那么这个匹配值最低的数一定是a1&a2&….&an,然后对于其他位上的数,可以进行全排列,理清楚了思路,代码实现就比较简单.

Problem - C - Codeforces

题目大意:现在对于一个仅由数字,规定一种这样的加法,对于该数字的每一位,都加上1,如果当前数字为9,那么就变为10,例如1912 + 1 = 21023,现在给定一个这样的数字,求出它加上k次之后得到的数字的位数.

思路:一道递推题,因为k的输入规模是2e5,所以可以直接模拟出k的每一种情况,先用一个数组num存下当前0~9每个字符的个数,每次+1之后,字符的个数依次向前移动一位,1的字符个数额外的加上之前9的字符个数,然后根据这个关系递推即可,因为面临多组输入,所以需要预处理,并将每个k的值存入数组dp,dp的意义是当前数字经过+m之后得到的长度,最后对于每个输入的数字n,将n的每一位数字拿出来进行运算.(需要优化读写)

Problem - D - Codeforces

题目大意:给定n个节点,和一个值p,每两个相邻节点存在一条边权为p的边,现在如果满足gcd(a1, a1+1,ai+2, …, aj) = min(a1, a1+1,ai+2, …, aj),那么i和j之间就存在一条边权为gcd(a1, a1+1,ai+2, …, aj)的边,现在对于整张图求其最小生成树.n<=2e5

思路:首先不能直接套用最小生成树的模板来做,因为n的值较大,这个图是根本建不起来的,但同时又需要有一种数据结构来表示连通关系,所以这里可以采用并查集来做.然后是如何考虑边权的问题,对于gcd(ai, ai+1,ai+2, …, aj) = min(ai, ai+1,ai+2, …, aj),如果满足该式子,那么ai就是最大质因数,其他所有能和它连边的节点值,都应该是它的倍数,然后对于这样的一个点,我们可以向左右两边分别进行连接,如果ai小于p,那么一定优先采用这种方式建图.为了降低复杂度,我们对原输入进行一个排序,先从较小的数值开始向两边举.当完成了gcd的建图之后,对于每相邻的两个节点,如果不连通,那么用边权为p的边再将其连上.
