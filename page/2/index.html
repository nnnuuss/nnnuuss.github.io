Problem - A - Codeforces

题目大意:给定两个数A和B,你可以采取两种操作,使A/=B,或者使B+=1,请输出最小的操作次数,使得A变为0

思路:看似很麻烦,实际上简单分析一下,如果要进行+1操作,那么应该最先进行这种操作,具体操作多少次,设一个界限然后依次枚举就可以了,然后取最小的总操作次数.

Problem - B - Codeforces

题目大意:现在规定两个序列k-similar符合以下条件

  1. 两个序列都是严格升序的.

  2. 两个序列长度相等.

  3. 两个序列中的元素都不超过k.

  4. 两个序列中只有一个数不相同.

现在给定你一个序列a,每次询问a的一个子序列,请求出有多少个序列和a的子序列k-similar

思路:首先一点,只有一个数不相同,且严格升序,当这个不同的数是原序列中最小的值的时候,它就可以在区间[1, a2-1]上除该点的位置任意选取,同理当这个数是最大数的时候,他就可以在区间[an-1+1, k]上除原位置之外的其他位置上选取,当一个点处于两点之间的时候,那么它就只能在两点之间来进行移动了.

分析完基本情况之后,我们可以用前缀和来统计一下处于两点之间的这些点的贡献值,然后对于每次查询直接计算即可.

Problem - C - Codeforces

题目大意:给定一个数a和b,请你找出有多少对(x,y),使得x/y==x%y.(1<=x<=a, 1<=y<=b).

思路:因为x,y的输入规模都是1e9,所以这题不能直接枚举,需要找规律来进行优化.

因为x / y = k && x % y==k,那么x=(y+1)k

因为k<y, 所以 x > y(y+1), x^1/2 > y > k.

经过这样的一系列证明我们发现k的数据范围降成了1e5左右,那么我们就可以枚举k来求解了.

关于如何枚举k,我们直接计算出a/k的大小,这个计算出来的就是理想下最大的y+1,如果y大于了b,那么y只能为b,对于每个y+1都可以在a的范围内找到对应的x,然后因为k<b,所以有效的b的个数是b-k个,

Problem - A - Codeforces

题目大意:我们定义函数S(x) = 数字x每一位上的数字之和,当满足S(x+1) < S(x)时,我们称数x是一个有趣的数,现在给定你一个数n,请你求出从1到n有多少个有趣的数.

思路:思维题,当最后一位出现9的时候,那么该数一定是一个有趣的数,有多少个每10个数个位上会出现一次9,那么只需要将n/10即可,需要注意的是,当n的个位上是9的时候,也是有一个有趣的数,所以应该计算(n+1)/10

Problem - B - Codeforces

题目大意:给定一个字符串s,和一个字符串t,你可以从任意一点开始从左向右遍历字符串s,然后在遍历过程中的任意一点开始改变方向,从右向左遍历,请问对于字符串s是否存在这样的一种遍历方式使得遍历结果等于字符串t.

思路:先将字符串s和t依次进行匹配,对于每一个可以匹配上的点,都应该尝试改变方向来进行遍历,这样才能确保所有的情况都被列举到.

Problem - C - Codeforces

题目大意:两个人交替参加比赛,共10场,当每场比赛的结果是1的时候就代表参加这场比赛的人获得1分,如果是0则不得分,如果是?则表示待定,现在两人进行比试,请你站在裁判的角度考虑至少比到第几场时,不需要再比了.(至少的意思是:如果当前两人的分差无法通过后续的比赛来进行弥补,那么就不需要再比了,同时你也可以使得?变成0或1来尽快结束)

思路:首先是个对于比赛结果不确定的情况,我们只由两种取法

一种是让奇数场次的不确定场全部获胜,偶数场次的不确定场全部失败

一种是让奇数场次的不确定场全部失败,偶数场次的不确定场全部获胜

只有这样分差才尽可能地大,然后对于结果的判断,只需要统计两个人的分数,然后判断当前的分差是否大于剩余场次能能拉平的分差,以此来判定是否提前结束比赛.

Problem - D - Codeforces

题目大意:你可以操控键盘打出一串预定字符,但在你准备输入一个字符时,你可以选择输入该字符,或者按下删除键(这样你不仅不会输入该字符,还会删掉之前存在的一个字符),现在给你一串预定字符,和一串目标字符,请问你是否可以打出目标字符串.

思路:思维题,首先如果目标串长度大于初始串那么一定不可能.

然后需要讨论两种情况

  1. 目标字符串和源字符串的差值为奇数.
    因为每次删除只由在开头的时候删除,才能删除一个字符,所以如果差值为奇数,那么一定是在开头就进行了一次删除.然后情况就变成了偶数情况

  2. 差值为偶数
    在后续的删除中每次删除就相当于把字符串同时删去了两位,所以对于这种情况只需要线性匹配一次就可以了.

得知这两种情况之后,其实只需要对两个串进行一次匹配,如果失配的话,直接将源字符串左移两位即可,相当于在某个字符失配之后直接进行删除.

Problem - D - Codeforces

题目大意:给定n个节点,和一个值p,每两个相邻节点存在一条边权为p的边,现在如果满足gcd(a1, a1+1,ai+2, …, aj) = min(a1, a1+1,ai+2, …, aj),那么i和j之间就存在一条边权为gcd(a1, a1+1,ai+2, …, aj)的边,现在对于整张图求其最小生成树.n<=2e5

思路:首先不能直接套用最小生成树的模板来做,因为n的值较大,这个图是根本建不起来的,但同时又需要有一种数据结构来表示连通关系,所以这里可以采用并查集来做.然后是如何考虑边权的问题,对于gcd(ai, ai+1,ai+2, …, aj) = min(ai, ai+1,ai+2, …, aj),如果满足该式子,那么ai就是最大质因数,其他所有能和它连边的节点值,都应该是它的倍数,然后对于这样的一个点,我们可以向左右两边分别进行连接,如果ai小于p,那么一定优先采用这种方式建图.为了降低复杂度,我们对原输入进行一个排序,先从较小的数值开始向两边举.当完成了gcd的建图之后,对于每相邻的两个节点,如果不连通,那么用边权为p的边再将其连上.

Problem - A - Codeforces

题目大意:给定一个长度为n的序列,你可以向序列中添加任意个非负数,来使得序列的平均值为1,求最小添加的个数.

思路:非三类讨论

  1. 当前序列中的所有数的和为负数或者是小于长度n,那么只需要添加一项即可.
  2. 当前序列中的所有数的都等于1,那么则不需要添加任何数,
  3. 当前序列中的所有数之和大于长度n,那么这时应该只添加0,并且每一次添加都可以使得当前序列长度加一,当添加到该序列长度为所有数之和时即可.

Problem - B - Codeforces

题目大意:给定一个平面,和一个人的初始坐标,你可以设置两个位置,来让其前往,请通过合理的设置来使得该人的行走距离尽可能地大,输出两个位置的坐标.

思路:首先该人到达两个位置的距离分别是abs(x - x1) + abs(y - y1) + abs(x - x2) + abs(y - y2).为了使绝对值尽可能地大,则应该将两个位置设在对角线上,这里有一个小坑,对角线有两组,你可能会因为当前人和某个角落重合而去选择另一组对角线,但是该位置可以和人的初始位置重合,所以无论哪种情况只需要输出任意一组即可.

Problem - D - Codeforces

题目大意:给定一个数x,Alice和Bob可以对这个数进行这样操作:选取这个数的一个因数k(k != 1 && k != x),然后将这个数变为x-k,当两个人谁无法进行该操作的时候,他就输了,假设两个人都会采用最优策略.Alice永远先手.

思路:博弈论,首先有几个关键的结论需要推出

  1. 如果当前数为奇数,那么该数的所有因数一定是奇数,所以无论怎么操作,该数一定是被操作偶数次,那么先手一定失败.
  2. 如果当前数是偶数,且可以通关变化变成奇数,那么状态就转移到了上一步,此时这种情况下先手一定胜利.
  3. 如果当前数是无法变成奇数的偶数,那么它的因数一定都是2,那么当2的个数为偶数时,可以进行奇数次操作,先手获胜,为奇数时进行偶数次操作,后手获胜.

以上所有的情况讨论完毕,只需代码实现即可.

Problem - E1 - Codeforces

题目大意:一个初始字符串a,和它的长度n,你可以对字符串进行切割从前向后得到一个子串,也可以对字串进行拼接,来使其变成长度增加.你需要通过一些的操作来使得a变为长度为m的字符串.请策划出一种最优的策略使得变换后的字符串字典序最小.

思路:首先我们需要明确的是如果想要变换后的字符串最小,那么该字符串一定是全由初始串的最优子串进行拼接而成的.因为假如最优的目标串是由aabaaa拼接而成的话,那么b是一种更优的结构,就不存在当前这种字符结构,所以该结论成立.

对于esay和hard版本最大的区别就是n的长度,当n较小的时候可以直接枚举所有的子串情况,然后操作之后进行比较,但是当n较大时显然无法通过,那么关键点就是如何求最优子串.

我们可以设一个最优字串的长度为k,然后将原串在下标i处与最优串对应区间数值进行比较,如果当前下标的字符大于最有区间的对应字符则结束,如果小于则将长度增加为当前位置,如果相等则不需要处理,等待后续匹配即可.

Problem - F - Codeforces

题目大意:给定n个节点和m条边,每个节点有自己的当前值,和目标值,你每次可以将一条边上的两个节点同时加上一个数k,求经过若干次变换后,所有节点上的当前值能否变为目标值.

思路:我们将每个结点的目标值减去当前值成为该节点的权值.

首先要明确一个的前提就是,每次进行加上k的操作之后,权值和一定是加减一个偶数,因此如果整张图的权值和是奇数的话,那么一定是无法变换的.

然后这个图需要分两类情况来讨论.

一类是比较特殊的情况,二分图,这时我们需要对整个图进行染色,此时改变一个节点的权值的同时也改变了另一种颜色的节点的权值,如果此时两种不同颜色节点的权值和不同的话,那么一定无法通过变化使得所有的节点权值都为0.

二类是另一种情况,即相同颜色的节点连在了一起,那么节点值的更新可以在同种颜色内部进行更新,那么此时就不需要多考虑什么,一定是合法.
