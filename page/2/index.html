Problem - A - Codeforces

题目大意:给定n个字符串,你可以从其中一个字符串中拿走一个字符并加到另一个字符串上,该操作可进行若干次,请问是否可以通过操作使得每个字符串都相等.

思路:如果要使每个字符串都相等,那么每个字符串中对应的每个字符个数都应该相等,只需要检验每个字符的个数是否可以被字符串个数整除即可.

Problem - B - Codeforces

题目大意:给定长度为n的序列a,你可以对该序列进行重新排序,现在要使得序列a满足这样的一个关系ai= c^i,即使得序列a中每一项都是c的次方,你可以对序列a中的每一个数进行加一或者减一,请求出最小的操作次数.

思路:暴力+技巧,因为这题让我们输出的答案没有取模,所以可以确定的是这个答案不会爆long long,那么我们直接从1开始枚举c,并用double来进行存储,如果枚举出的序列c中的某一项爆了long long,那么该枚举过程就可以中止了,因为后面必然也会爆.

Problem - C - Codeforces

题目大意:给定长度为n的序列a,你每次可以选择长度为k的连续子序列,对于子序列中的每一个元素,它可以加上k的任意倍数,请你对序列a操作恰好3次来使得序列a中的每个元素都为0.

思路:比较好想的构造题,我们可以先想办法让所有的项都变成n的倍数,最后再对所有数减去n的倍数,所以我们先取前n-1个数,然后使每个数都变成n的倍数,因为n和n-1相差一,所以加上若干个n-1是一定可以改变余数的,然后在对第n个数进行操作,然后对所有数相减即可,需要注意的是,这里需要对n=1的情况进行特判.

Problem - D - Codeforces

题目大意:给定n堆石子,A,B两个人每次可以选择一堆石子从中取出一颗,上一轮被取到的那一堆石子,当前轮不可以被取到,当一个人无法取石子时,该人失败,请判断谁必胜.

思路:博弈论.对于这类题目需要从最简单的基础状态讨论起,然后利用基础状态得到递推.

最简单的状态就是如果一堆石子大于其他堆石子的总和,那么先手A直接选择这堆石子,该玩家一定获胜.

如果不满足必胜状态,那么先手玩家为了使自己达到必胜状态,会选择其它较小堆的石子,来使得除较大堆石子的总和减小,而后手玩家为了避免其获胜,就会选择较大堆的石子,那么根据该种情况不断递推下去,如果石子总和是奇数的话,先手玩家一定有办法使得自己手上持有一子,而偶数情况下,一定无法获胜.

Problem - A - Codeforces

题目大意:给定你一个点a的坐标,现在你可以在区间[0,a]上放一个点b,使得OA,AB的距离差为k,如果当前情况下无解,可以选择讲a点移动几位,请求出最小移动次数(当前有解则不需要移动).

思路:思维题,先考虑k大于a的情况,这种情况下唯一的办法就是右移a,移动的最小距离就是k-a,第二种情况,k<=a,如果a为偶数的话,那么距离差只能为偶数,如果a为奇数的话,距离差只能为奇数,如果a,k的奇偶性不同,那么就必须移动一格.

Problem - B - Codeforces

题目大意:给定序列a和序列b,两个序列中的元素都为0,1,2三种,我们这样定义序列c请安排序列a,b的顺序,使得序列c的和最大.

思路:思维题,因为序列中的元素只有0,1,2,所以情况其实很好讨论,如果要使序列和增加,只能是序列a的2和序列b的1进行配对,使序列和减小只能是序列b的2和序列a的1配对,那么为了使总和尽可能地大,就要让序列a的2尽可能地和序列b的1进行配对,同时尽可能地避免序列b的2和序列a的1进行配对.

Problem - C - Codeforces

题目大意:如果一个序列中的两个元素的gcd等于该序列中的最小元素,那么这两个元素可以进行位置交换,现在给定一个序列,请判断该序列是否可以通过交换来变成一个非严格升序的序列.

思路:思维题,通过观察可以发现,如果几个数的gcd等于序列中的最小元素,那么这几个数和序列中的最小元素是可以任意交换位置的,那么我们只需要先将原序列排个序,如果某个数排序之后不在之前的位置上,那么它就应该参与交换,最后整理出一个需要交换的数的集合,如果这些数都满足任意交换的条件,那么就是可行的.

Problem - D - Codeforces

思路:比较简单的思维题,因为大于2的质数一定是奇数,那么它对于2的余数一定是1,然后对于n,它和n-1互质,与n-1的余数也是1,所以只需要输出2和n-1即可.

请问Alice最多有多少颗棋子可以移动到最上方一行.

Problem - C - Codeforces

  1. 让某两个人之间建立友谊关系.
  2. 清除某两个人之间的友谊关系.
  3. 计算当前的存活人数,计算规则如下,如果一个人的权力低于他的每个朋友,那么该人被清除,该过程会一直进行下去,直到停下为止,输出停下后还存活的人数.(清除会从权力最小的人开始清除,不是同时清除)

Problem - D - Codeforces

题目大意:如果一个序列满足,对于序列中的每一个数,存在一个数m,满足ai%m都相等,那么我们称这样一个序列是一个good序列,对于一个长度为n的序列,请你求出它的子序列中good序列的最大长度.

思路:因为要求ai%m相等,ai = m*j + i, i是余数那么我们只需要考虑m*j即可,实际上就是求区间上每个m*j的gcd,那么我们需要做的就是用差分数组维护一个gcd,然后再枚举左端点,因为gcd满足单调性,对于区间长度固定后,如果增加区间长度,gcd只可能变小,而减小区间长度,gcd可能变大,所以对于右端点可以采用二分来选取.这样复杂度就是nlogn+nlogn.

思路:一道贪心+模拟,首先我们从前向后遍历一次队伍耗时,如果在某处不满足非严格升序,那么该位置的队伍一定是比钱一支队伍少做一道题,然后这样遍历一次,如果当前最后的名次的队伍解决问题的数目刚好是0或者1(为1时代表当前所有队伍都做出了题)那么就是符合的否则就是不符合的,同时这里需要加一个特判,当所有人耗时都是0的时候,就是大家都没过题,那也是确定的.

思路:贪心+bfs,因为楼层之间互相连通,那么我们每次应该直接从最高的楼栋开始考虑,然后根据连通性,高楼层如果能抵达,那么与它相连的更低的楼层也可以抵达,然后对所有的点跑一遍bfs,就可以得出答案.

Problem - B - Codeforces

题目大意:现在规定两个序列k-similar符合以下条件

  1. 两个序列都是严格升序的.

  2. 两个序列长度相等.

  3. 两个序列中的元素都不超过k.

  4. 两个序列中只有一个数不相同.

现在给定你一个序列a,每次询问a的一个子序列,请求出有多少个序列和a的子序列k-similar

思路:首先一点,只有一个数不相同,且严格升序,当这个不同的数是原序列中最小的值的时候,它就可以在区间[1, a2-1]上除该点的位置任意选取,同理当这个数是最大数的时候,他就可以在区间[an-1+1, k]上除原位置之外的其他位置上选取,当一个点处于两点之间的时候,那么它就只能在两点之间来进行移动了.

分析完基本情况之后,我们可以用前缀和来统计一下处于两点之间的这些点的贡献值,然后对于每次查询直接计算即可.

思路:因为x,y的输入规模都是1e9,所以这题不能直接枚举,需要找规律来进行优化.

因为x / y = k && x % y==k,那么x=(y+1)k

因为k<y, 所以 x > y(y+1), x^1/2 > y > k.
