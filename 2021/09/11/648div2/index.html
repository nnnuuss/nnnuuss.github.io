Problem - A - Codeforces

题目大意:给定一个n*m的棋盘,如果某个格子的一行和一列都是0,那么这个格子可以填1,A和B轮流在棋盘上填1,如果谁无法填那么他就落败,请判断两个人的胜负.

思路:直接暴力模拟.

Problem - B - Codeforces

题目大意:给定n个数,每个数有两个属性,一个是它本身的值,一个是它的类型,类型分为0和1,你每次可以将类型不同的数字进行交换,请问你是否可以通过交换使得序列变成升序排列.

思路:思维题,讨论即可,如果如果该序列中同时存在0和1,那么所有的数字都可以进行交换,否则的话就无法交换,需要判断当前序列是否有序.

Problem - C - Codeforces

题目大意:给定一个排列a和一个排列b,你可以对b中的每个元素左移k位,第n个元素左移一位后变成第一个元素,请你求出经过平移之后最多,序列a中最多有多少个数字和b是匹配的.

思路:找出序列b中每个元素和序列a中元素对应之后的距离,然后距离相同的分为一组,因为距离相同的元素通过平移若干位后一定能够能序列a对上,然后线性的扫一次求个最大值即可.

Problem - D - Codeforces

题目大意:给定你一张大小位n*m的地图,地图上有四种标志,空,强,好人和坏人,地图的出口为右下角,你可以将某些空的位置变成墙,请问你是否有一种构造方式使得所有的好人都能够顺利逃出,所有的坏人都无法逃出.

思路:思维+暴力.对于每个坏人,他四周的位置能变成墙的都应该变成墙,然后再搜索一次,检查每个好人是否都能逃离,同时也要判断是否每个坏人都无法逃离,然后就可以得到答案.

Problem - E - Codeforces

题目大意:给定你一个长度为n的序列,你可以从中选取k个元素作为子序列,对于这k个数,如果它们在二进制的表示的第i位上的1等于max(k-2,1)那么这个序列的值ans+=2^(i-1),请求出所给序列的最大值.

思路:思考一下可以得知,当k=3的时候是最优解,然后数据量比较小,可以直接冲,但我觉得正确做法应该是通过a|b=a,来将b进行去重,从而来减小n的个数降低复杂度.

Problem - F - Codeforces

题目大意:给定你长度为n的序列a和序列b,你每次可以将序列a的长度为k的前缀和长度为k的后缀进行交换,请你判断是否可以通过若干次交换使得序列a变成序列b.

思路:构造题,这题需要找到一个规律,我们可以发现呈轴对称的两个元素在交换之后是不会改变的,而且每一对元素可以通过交换出现在任意位置,只需要先从靠近对称轴的开始匹配,后面的元素都可以依次匹配,所以只需要判断是否每一对数字都能够配对上即可.
