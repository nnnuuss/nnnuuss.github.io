(初期)一,基本算法

1.枚举

Flip Game - POJ 1753 - Virtual Judge (vjudge.net)

题目大意:给定一个黑白棋盘,每次可以选择翻动一颗棋子,当一颗棋子翻转后,其四周的棋子都会进行翻转,求如何最少次数的翻转可以使得棋盘上的棋子都为黑或者白.
思路:因为之前的枚举第一行的情况不知道为啥不过,所以重新写了一个二进制枚举所有棋子状态的,第一行就表示0000,这样16个数字由一个16位数字表示,1表示进行翻转.

2.贪心

Radar Installation - POJ 1328 - Virtual Judge (vjudge.net)

题目大意:x轴下部为陆地,上部为小岛,可以随意在陆地上布置炮台,问最少使用多少炮台可以使得所有的小岛都在炮台的射程内.

思路分析:首先为了保证炮台的射程尽可能地大,我们的炮台一定要放在x轴上,这样才能确保y轴的距离尽可能地小.
现在分析y轴该如何确定,我首先想的是把所有的小岛以x轴的位置进行一个升序排序,然后每个炮台都与最左边对其,但这样的思路显然是错,例如如果这样选择先跟A对其,那么B就不在范围内了.
所以正确的做法是,逆向思维,确定每个小岛的炮台区间,然后求最小无交集区间个数.

先贴上别人的ac代码

Power of Cryptography - POJ 2109 - Virtual Judge (vjudge.net)

题目大意:给定两个数a,b,一定存在一个数n使得n^a = b,求得n,b的输入规模非常大为,10^101
思路:需要借用浮点数的表示规则,浮点数最大是可以表示2^26大小的数的,所以直接用浮点数存取,然后对b开a次方.

Y2K Accounting Bug - POJ 2586 - Virtual Judge (vjudge.net)

题目大意:一个公司以每次提供连续5月的账单(1,2,3,4,5 ; 2, 3, 4, 5, 6)来提供每年的账单,每次提供的账单一定是亏损的,求全年的收益是否能够结余,如果有亏随,每个月的亏损为d,结余为s.
思路:直接画图枚举情况即可,

二,最短路径算法

Currency Exchange - POJ 1860 - Virtual Judge (vjudge.net)

题目大意:给定你若干种货币,给定你一个交换规则,你可以通过手上的货币与其他货币进行交换,每次交换得到的货币为(已有货币-手续费)*汇率,求解是否可以通过交换后,使得手上的货币数量和原来的相同.
思路:是一个图的问题,问题经过分解之后,就是在一个起点开始,求整个图是否有负圈,如果有负圈的话,那么就可以使得货币不断地增加,最总抵消手续的费用,所以把问题转化一下,就是在有限次数内求最短路径,如果所求的次数超过了有限次,那么一定存在负圈,也就是答案成立.因为是单源图,所以可以直接采用bellman,直接更新到达每个点的最短路径,然后不断更新.

bellman的板子

Wormholes - POJ 3259 - Virtual Judge (vjudge.net)

题目大意:给定若干个节点,每个节点间有若干路径,有些路径是负值,判断是否可以从某一点出发,使得终值为负值.
思路:直接用bellman判断是否有负圈.

昂贵的聘礼 - POJ 1062 - Virtual Judge (vjudge.net)

题目大意:每样物品有它的价值,和优惠价值,优惠价值就是当你使用这件物品和别人交换物品时需要付出的价值,每件物品的所有者还有等级制度,每次进行交易的等级差值不能高于题目所给条件.求交换到第一件物品所需要的最小价值.
思路:dijk求最短路,不过因为是每次连续的交易不可以超过m,所以要对每个节点进行一次dijk,然后得出答案,建图比较辛苦.

Stockbroker Grapevine - POJ 1125 - Virtual Judge (vjudge.net)

题目大意:有若干个间谍,每个间谍可以和他的下家进行信息交流,并花费一定的时间,请求出最优路径,使得每个间谍的都可以接收到信息,并且花费的最差时间最短.
思路:Floyd求任意两点的最短路径,然后枚举每个间谍的情况,求出最小值.

Arbitrage - POJ 2240 - Virtual Judge (vjudge.net)

题目大意:给定若干个货币类型,和不同货币之间的汇率,求是否可以通过交换,使得手上的货币越来越多.
思路:既可以bellman求正圈,可以Floyd求从当前点到当前点的汇率,如果有汇率大于1,那么就存在正圈

五,动态规划

Cash Machine - POJ 1276 - Virtual Judge (vjudge.net)

题目大意:给定一定的货币金额和对应的数量,给定一个金额,求出所给货币所能组成的不超过所给金额的最大金额.

混合背包的模板题

混合背包类型问题,主要是对于每种物品,有数量限制,做法是直接把每种物品的若干数量展开,当作完全相同的不同物品来进行抉择,但是考虑到这样的时间复杂度直接从nw增加到nw∑k,时间复杂度过高,所以需要进行优化.因为nw的时间复杂度无法再继续优化,所以只能从每次抉择的物品上进行取舍,因为k1,k2和k2,k3效果是一样的,所以我们采取二进制分组的方式,使数量由2倍增,及1, 2, 4, 8…..然后最后剩下的一部分又作为一组进行抉择,这样就把k相关的复杂度降低到log了.

The Cow Lexicon - POJ 3267 - Virtual Judge (vjudge.net)

题目大意:给定一串字符,和一个字典,求将当前字符串最少除去多少个字符可以使得当前字串可以全由字典中字符串组成.

思路:dp[i]的意义表示从第i下标开始到末尾构成的字符串去除多少字符后可以与字典中字符匹配.递推关系式为dp[j] = min{dp[j], dp[i] + W},从后往前遍历,每次当可以对字典中进行匹配的时候进行更新,具体细节看代码实现.

Human Gene Functions - POJ 1080 - Virtual Judge (vjudge.net)

题目大意:给定一组dna序列,你可以对其中较短的一组dna序列进行添加’-‘来使得两个dna进行配对,保证最大的配对值.

思路:有点像编辑距离,状态转移方程是一样的

唯一不同的一点是,每次匹配一个字符时,需要根据对应的匹配值表来取出结果.当当前两个字符不相等时,直接取a串用’-‘匹配和b串用’-‘匹配的最优解,最不好理解的一点就是这个方程.

当当前两个字符相等时,这时两个字符同时向前移动一位,它们的上一个状态就是i-1和j-1,当a串也就是i对应的串前进一位时,则选择让前一个匹配到该点位置的状态

对于dp[1][1]而言,此时对应的a串位置字符为’A’,B串时’G’,如果选择将a串的字符用’-‘进行匹配,那么所需要的上一个状态就是G先和’-‘配对,然后得到这样的一个匹配结果G-和-A,大概就这样,也不是很能说清.

dp数组的定义是对于a串前i个字符和b串前j个字符进行匹配得到的数值,如果当前不相等,那么如果将a串该字符用’-‘进行配对的话,就是需要将i-1前面的串和现有的b串进行匹配,所以是dp[i-1][j] +sum(‘-‘, s1[i]),然后运算的是当前a串字符的值,同理dp[i][j-1] +sum(‘-‘, s2[j]).
