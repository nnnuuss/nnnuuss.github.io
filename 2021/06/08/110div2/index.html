Problem - B - Codeforces

题目大意:给定一个序列,如果对于下标i < j满足gcd(ai, 2aj) > 1, 那么称这样一个数对为good,你可以对序列进行重新整理,保证good数对的数量最大.
思路:思维题,因为是ai和2aj,所以对于任意可以被2整除的数而言,它后面的数形成的数对都是一个good数对,所以思路就是先将偶数排成一队,然后再将奇数进行处理,奇数部分需要直接枚举来得到答案.这题主要思路就是先放偶数再放奇数.

Problem - C - Codeforces

题目大意:如果一个字符串仅由01组成,且相邻字符都不相同,那么就称这样一个字符串为不稳定串,如果一个字符串由01?组成,且通过将?替换成0或1的话,那么这样的一个串就称为beautiful串,对于给定的字符串,求出最大数量的beautiful子串.
思路:直接抄答案,还抄错了,多找了好几篇才看懂.
首先有这么几点性质需要先注意:
1.任何beautiful串的连续子串都是beautiful串.
2.任何非beautiful串的母串都是非beautiful串.
因为输入规模为2e5,所以必须使用线性方式,对于此类求子串的问题,一般是直接枚举左右端点,然后进行判断.

r - l + 1;

我们需要枚举左右端点,如果当前构成一个beautiful串的话,那么首先加上当前字符本身作为一个单独的串,然后加上从左端点到右端点之前的每个左端点连接而成的子串.
但是如何对这个最长连续串进行维护,我们需要考虑非法的情况,其实如果希望一个串是beautiful串,那么它必须和101010或者是010101完全相符,?是可以自动适配的所以不需要考虑,我们换种说法就是必须奇数位上出现的全是一种元素,偶数位上出现的全是另一种元素,隐居这个逻辑,我们写出代码.
