Problem - A - Codeforces

题目大意:给定一个长度为n的序列a,你可以重新安排序列,使得的值为m,判断是否存在这样的一个顺序.

思路:观察之后可以发现,序列的值和顺序无关,所以只需要判断a1+a2+….+an的值是否等于m就可以了.

Problem - B - Codeforces

题目大意:请你构造一个大小为n*n的矩阵,满足矩阵中的每个数字都是非质数,且每一行每一列相加都是质数.

思路:简单构造题,我们考虑除了斜向的一列,其余部分全部填1的情况,这种情况下,只需要找到一个数c,c-n+1不是质数,且c是质数就可以了.

Problem - C - Codeforces

题目大意:按照图示的二分搜索程序,查找一个元素x,且x的下标为pos,请计算出存在多少个序列可以在上图的程序下在pos位置找到x.

思路:组合数学+模拟,因为二分搜索实际上只需要保证每次二分出的那几个数跟目标值保持正确的大小关系即可,那么对于目标值左边的部分,只需要计算出被二分到的值k,然后从左边的元素里选取k个数,然后将它们随意排列,同理右边,除了这些位置上的数和目标数,其他位置上的数可以随意排列.

Problem - D - Codeforces

题目大意:给定一颗有n个节点的数,节点1为根节点,每个节点的权值可以分配给它的所有子节点,请计算出所有叶节点的可能最小值的最大值.

思路:二分答案,对于每个二分出的答案,对叶结点进行处理,如果当前叶子本身的权值大于m,那么就是有多余的权值无法分配,当前情况就是不合法的,如果合法,更新当前达到答案还需要的权值并向上传递,对于非叶子节点,如果当前节点的权值大于所有子节点需要的权值,那么同样也是多余的权值无法分配,所以也是不合法的,然后根据这个原则不断二分就可以了,唯一要注意的就是对于某组数据,溢出了,需要特殊处理一下.
