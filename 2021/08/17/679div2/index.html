Problem - A - Codeforces

题目大意:给定长度为n的序列(n为偶数),请你构造一个序列b使得a1*b1 + a2*b2 + …

+an*bn=0.

思路:因为n是偶数,所以每次选取相邻的两位,然后求他们的最小公倍数c,使得ai*bi = c, ai+1*bi+1 =-c即可.

Problem - B - Codeforces

题目大意:给定一个大小为n*m的方阵,现在按照随机顺序给出每一行,又按照随机顺序给出每一列,请你根据这两个量来还原方阵.(保证方阵中每个值都不想等)

思路:因为方阵中没有重复元素,那么我们只需要对按列给出的第一组数据进行处理,因为按列给出,那么列中每一个元素的顺序一定是符合行的顺序的,只需要从第一个元素开始,遍历所有元素,然后找到每个元素所对应的行就可以得到答案.

Problem - C - Codeforces

题目大意:现在给定6根琴弦,每根琴弦有自己的权值,又给定若干个音符,每个音符都是由琴弦所弹奏出的,音符的权值bi = ci+aj,(j表示琴弦的下标),请求出cmax-cmin的最小值.

思路:双指针维护区间.为了使cmax-cmin尽可能的小,那么我们只需要将所有的c都存储起来,然后双指针来维护一个区间,保证区间内的所有的音符都出现过即可,然后线性的扫一遍来更新答案.

Problem - D - Codeforces

题目大意:给定一个数n,然后进行2*n次操作,操作有两种一种是’+’,它的意义是表示从1~n的数字里面选择一个加入到集合中,还有一种操作是’-‘ num,它表示从当前的集合中取出num,且num是集合的最小值,现在给定一系列操作,请判断该操作是否有一种符合的元素添加顺序,使得整个操作合法.

思路:模拟.正向模拟的话会比较麻烦,这里选择反向用优先队列来维护,对于最后出队的元素,我们把它先加入队列中,如果当前的加入的元素大于当前队列的最小值,那么一定是不符合的,如果当前的操作是加入一个元素,那我们优先考虑队列的最小值,将它加入队列,因为该值小于队列中的其它元素,如果从集合中取出,也是它被优先取出,所以应该延缓插入.
