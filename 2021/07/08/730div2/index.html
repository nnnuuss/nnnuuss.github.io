Problem - A - Codeforces

题目大意:给定两个数,你可以对两个数进行两种操作,两个数同时加1或者两个数同时减1,求两个数经过若干次操作后的gcd,并求出最小的操作次数.

思路:无论进行多少次操作,两个数的差值都不会改变,那么这个差值就是两个数的gcd,然后只需要使得两个数进行操作后都是差值的倍数就可以了,至于进行多少次操作,只需要求出分别讨论加1和减1的情况,然后取最小值.

Problem - B - Codeforces

题目大意:给定若干个车道,对于车道上的车辆有一定的阻塞值,阻塞值的算法为当前车道上车数-该车道以后每个车道上的车数的绝对值之和,你被允许进行若干次的车辆交换,使得车辆到达不同的车道上,求解最小阻塞值.

思路:为了使阻塞值尽可能地小,那么就应该使得每个车道上的车辆尽可能地数目接近,那么其实问题直接分解成了一种情况,任意两个车道上的车辆数目差值不是1就是0,那么最小阻塞值就是当前为1的数目乘上当前为0的数目.

Problem - C - Codeforces

题目大意:给定三个字符cmp各自的概率,和一个值v,每次可以选定其中一个字符,当选中的字符为P时不可再选取其他字符,当选取一个字符后,取当前该字符的值和v的较小值temp,然后该字符的概率减少temp,其概率平分到剩余的字符上,如果某个字符概率为0,那么概率不可以分给它,最终抽取的形成的一个字符串,字符串的长度乘以当前的选取的概率就是该种情况的贡献值,求所有可能情况的贡献值之和.

思路:模拟+爆搜,其实翻译出来题目大意就可以直接写了.

Problem - D1 - Codeforces

题目大意:现定义了一种不进位的加法,用于运算k位制的两数相加,easy版本k==2,二进制的不进位加法,即异或操作.给定一个密码x,其范围在[0,n-1]之间,每次你可以进行一次猜测,如果猜测值正确,那么会输出一个1,否则会输出一个0,并且原密码x变为x+y(这里的+表示不进位加法),最多猜测n次.

思路:对于二进制下的操作,所有的加法都可以进行异或来代替,那么只需要从0开始枚举,我们只需要使得每次密码x变换时,猜测的y也跟着变换就可以枚举出来了.

设当前猜测的密码为y 初始密码为x

第一次 y = 0 如果错误当前密码z=x+0

第二次 y = 0+1 如果错误当前密码z=x+0+0+1 = x+1

第三次 y = 1+2 如果错误当前密码z=x+1+1+2 = x+2

第n次 y = n-2+n-1 上一次变换后的密码z=x+n-2
