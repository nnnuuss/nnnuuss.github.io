C-Corrupted Contest_Benelux Algorithm Programming Contest 2020 (nowcoder.com)

题目大意:有n个队伍参赛,每个队伍的用时是他们解决的所有问题中最长的一道,队伍的排名先参考过题数,如果过题数相同,用时短的成绩更优,现在按照队伍的名次给定每个队伍的耗时,和比赛总共的题目个数,请确定每个队伍解决的题目个数是否确定.(耗时0的队伍代表一道题都未解决)

思路:一道贪心+模拟,首先我们从前向后遍历一次队伍耗时,如果在某处不满足非严格升序,那么该位置的队伍一定是比钱一支队伍少做一道题,然后这样遍历一次,如果当前最后的名次的队伍解决问题的数目刚好是0或者1(为1时代表当前所有队伍都做出了题)那么就是符合的否则就是不符合的,同时这里需要加一个特判,当所有人耗时都是0的时候,就是大家都没过题,那也是确定的.

D-Efficiently Elevated_Benelux Algorithm Programming Contest 2020 (nowcoder.com)

题目大意:给定若干个楼栋,每个楼有自己的高度,相连的楼栋之间同楼层是互相连通的(对角线不连通),现在要给楼栋安装电梯(电梯可以最高抵达它所位于的楼栋顶层),使得每个楼的每一层都可以抵达,请求出最小安装的电梯数.(如果一栋楼高度为1,那么不需要安装电梯)

思路:贪心+bfs,因为楼层之间互相连通,那么我们每次应该直接从最高的楼栋开始考虑,然后根据连通性,高楼层如果能抵达,那么与它相连的更低的楼层也可以抵达,然后对所有的点跑一遍bfs,就可以得出答案.

H-Incomplete Implementation_Benelux Algorithm Programming Contest 2020 (nowcoder.com)

题目大意:给定你一个长度为n的序列,你每次只能将其中的n/2个元素重新排序,你最多进行三次这样的排序,请输出每次进行排序的元素的下标,来使得三次排序后序列有序.

思路:思维+模拟,因为最多排三次,而且没有要求求最小次数,那么我们对于每一次排列,直接让n/4个元素归到自己的位置上即可,那么3此排序后有3n/4的元素归位了,整个序列也就有序了,关于每次该让那些元素归位,我这里写的是先将较大的元素归位,具体细节见代码.
