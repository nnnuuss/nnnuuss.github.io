Problem - A - Codeforces

题目大意:给定一个数n,你可以选择两个不相等的数a,b,满足(1$\leq$a<b$\leq$n),请你求出最大的GCD(a,b).

思路:不需要想太麻烦,最小的素数是2,那么a和b一个是n可以选择的最大的偶数,一个是偶数/2.只需要特判一下2就可以,因为2和1的gcd为1.

Problem - B - Codeforces

题目大意:给定2n个正数,你每次可以选择两个数进行合并,然后将所有合并的元素作为一个序列,请给出一种合并的方式,使得序列种n-1个元素的gcd不为1.

思路:其实思路很简单,还是最小素数为2的考点,我们只需要将奇数放在一起合并,偶数放在一起合并,那么一定满足这些元素的gcd>=2.

Problem - C - Codeforces

题目大意:给定一个数n,Alice和Bob每次可以对这个数进行两种操作

  1. 将当前数除以该数的一个奇因数.

  2. 将当前数减1(该数为1时不可再进行操作).

Alice先手,如果谁无法进行上述操作了,那么该玩家失败.

思路:找规律,如果当前数是奇数,那么可以直接将该数除以它本身,所以当前数是奇数,先手一定获胜.

如果当前数是偶数,非三类讨论,如果当前数是2,那么一定后手胜,属于一个特判.

如果当前数的因数全为偶数,那么先手只能进行操作2,后手状态转移到奇数状态,后首胜.

如果当前数的因数有奇数,那么先手的赢法就是将该数的变换成非2的偶数,这样后手只能进行操作2,回到奇数状态,先手获胜.

Problem - D - Codeforces

题目大意:现在规定一个序列的值k为min(max(s1, s3, s5,…,s2i-1), max(s2, s4, s6,…,s2i)).现在给定一个序列,你可以对它取任意子序列来使得k值尽可能地小.

思路:二分答案枚举出答案,然后用对于每一个答案,采用模拟的方式来检验.

遇到这种比较麻烦的构造+贪心,可以考虑一下采用二分答案+模拟的方式来尝试一下,不过要注意数据的范围,不能太大.
