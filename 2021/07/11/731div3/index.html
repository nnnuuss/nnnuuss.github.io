Problem - A - Codeforces

题目大意:给定起点的坐标和终点的坐标,还有一个障碍物的坐标,求出从起点移动到终点的最短距离(只能横向纵向移动).

思路:首先最短距离肯定之终点起点的坐标只差的绝对值之和,然后要考虑障碍物的问题 ,如果障碍物和起点终点都在同一条直线上,而且障碍物在起点终点之间,那么就需要绕路,其他的情况都可以直接走过去.

进行判断相等的时候不能进行连登(哭哭) a==b==c是得不到正确答案的

Problem - B - Codeforces

题目大意:给定一个字符串,问当前字符串是不是一个Alphabetical串.Alphabetical串从指字符’a’开始,然后左右两边任意的按字母顺序来添加字符得到的串.

思路:直接双指针模拟一下就可以了,比较直接的想法事先确定起点’a’的位置,但这样其实太麻烦了,还需要很多特判,所以我们换一个思路,让左右指针依次往中间移动,这样代码更加简洁.

Problem - C - Codeforces

题目大意:两个人协同合作写一段代码,然后给定两个人的操作序列,你可以进行一定的安排,来尽可能地使得每个操作都可以被完成,但是安排过后的操作要符合原来每个人的操作顺序.当操作数=0时,就代表添加一行代码,当操作数>0时就表示修改第操作数行的代码,如果修改行不存在,那么操作就非法了,求解是否可以经过安排使得每个人的操作都合法.

思路:将操作数在满足各自顺序的基础上进行一个排序,然后直接依次判断.为了保证后续的修改,添加行的操作一定要尽可能地靠前.

本题出现的一些小细节:vector如果只push_back使用完后一定要进行清除.

Problem - D - Codeforces

题目大意:现在有一种新的升序规则,该规则规定,当后一个数与前一个数取&时,等于前一个数,就表示该数大于前数.给定一个序列,可以对给定的序列进行^操作,使得操作后的序列满足新升序规则,输出^的操作数.

思路:首先新的升序规则表示,较大数低位与较小数的所有位上应该相等,置于如何得出答案,我们只需要将理想的升序数与当前数进行^操作就可以,而理想数就是将前一个理想数的所有位全部复制到当前数上,对于第一个数,我们不需要进行^操作,它的理想数就是它本身.
